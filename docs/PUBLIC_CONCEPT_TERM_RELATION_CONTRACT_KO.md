# 공개 법률개념 용어 관계 계약

## 목적

본문의 법률용어 밑줄은 단순한 문자열 강조가 아니라 특정 개념카드로 연결되는 의미 관계다. 따라서 검색 편의를 위한 표현을 모두 같은 뜻으로 처리하지 않는다.

## 개념 하나의 기준

다음 중 하나라도 다르면 별도 개념카드를 만든다.

- 적용 대상이나 주체가 다르다.
- 성립 요건이 다르다.
- 법률효과가 다르다.
- 다른 개념의 상위·하위 개념이다.
- 같은 사람이라도 특정 관계나 절차에서의 지위를 가리킨다.

예를 들어 상속인, 법정상속인, 공동상속인은 관련성이 높지만 동일한 정의로 합치지 않는다.

## 검색 표현과 본문 자동 해설의 분리

- `aliases_ko`: 사용자가 함께 검색할 수 있도록 연결하는 표현이다. 같은 뜻이라는 보장은 없다.
- `term_relations`: 각 검색 표현과 대표 용어의 관계 및 공식 근거를 기록한다.
- 본문 자동 해설은 `exact_synonym`, `abbreviation`, `spelling_variant`만 사용한다.
- `plain_language`, `narrower`, `broader`, `related`는 검색 연결에는 사용할 수 있지만 같은 팝오버를 띄우지 않는다.
- 관계 분류나 공식 근거가 없으면 본문에 밑줄을 만들지 않는다.

## 생산 절차

1. 주제 콘텐츠의 법리·사실분기·질문·검색어에서 반복되는 어려운 표현을 수집한다.
2. 활성 룰링크 원장과 공식 근거 좌표로 의미와 사용 문맥을 확인한다.
3. 기존 개념과 완전히 같은지, 쉬운 표현인지, 상·하위 또는 관련 개념인지 분류한다.
4. 요건·효과·한계가 다르면 새 개념카드를 작성한다.
5. 검색 표현마다 `term_relations`와 `source_coordinate_ids`를 기록한다.
6. 출판 조립기가 근거 누락, 관계 미분류, 중복 자동 해설 용어를 차단한다.
7. 승인된 개념만 본문 팝오버·개념 페이지·검색에 반영한다.

## 축적 우선순위

다음 점수를 기계적으로 집계해 높은 표현부터 개념카드로 만든다.

- 공개 콘텐츠에서 등장한 서로 다른 글의 수
- 같은 글 안의 반복 횟수
- 법리 결론을 이해하는 데 필요한 정도
- 일반어와 법률상 의미가 달라 오해하기 쉬운 정도
- 이미 연결된 공식 근거 좌표의 유무

최종 정의와 관계 분류는 법률 담당자가 확인하되, 후보의 발견과 우선순위는 위 기준으로 반복 가능하게 산출한다.

## 금지사항

- 긴 표현이 짧은 대표 용어를 포함한다는 이유만으로 별칭 처리하지 않는다.
- 검색 유입을 늘리기 위해 근거 없는 동의어를 추가하지 않는다.
- 별도 개념이 필요한 표현을 기존 팝오버로 임시 연결하지 않는다.
- 팝오버가 없는 표현을 오류로 보지 않는다. 잘못된 설명보다 미표시가 우선이다.
