# 공개 법률정보와 변호사 전용 작업공간 운영 경계

## 1. 제품 결정

RuleLink는 하나의 화면에서 일반인 대상 법률정보와 사건별 법률서비스를 섞지 않는다.

- 공개 RuleLink: 누구나 무료로 읽는 법령·판례·일반 법리·생활상황·체크리스트
- 변호사 전용 작업공간: 구체 사건의 사실관계를 받아 결론·전략·증거·서면 방향을 검토하는 도구
- 직접 법률서비스: 변호사 또는 법무법인이 이용자와 직접 계약하고 명의·보수·책임을 부담하는 영역

면책문구는 이 경계를 대신하지 않는다. 데이터 흐름, 이용자 자격, 계약과 보수, 답변 명의와 책임을 실제 기능으로 분리한다.

## 2. 공개 사이트에서 허용하는 기능

- 공식 법령·판례 원문과 기준일 표시
- 일반적인 법리와 결론을 가르는 사실분기
- 사용자의 기기에만 저장되는 체크리스트
- 일반적인 준비자료·절차·주의사항
- 인쇄와 PDF 저장
- 변호사 전용 작업공간의 이용 이유와 자격 확인 절차 안내

공개 사이트는 이용자의 사건 내용을 수집하거나 작업공간으로 전송하지 않는다. 체크 표시도 서버나 다른 사이트에 보내지 않는다.

## 3. 공개 사이트에서 제공하지 않는 기능

- 이용자의 전체 사실관계를 받은 사건별 결론
- 승소 가능성·청구 또는 방어 전략
- 구체적인 증거수집·제출서면 작성 방향
- 회사 명의 또는 회사 책임의 법률답변
- 특정 변호사에게 사건을 자동 배정하는 기능
- 상담 성사·수임·성과에 연동된 소개료나 수익배분
- 공개 체크리스트를 사건요청 초안으로 자동 복사·전송하는 기능

## 4. 변호사 전용 게이트

자기선언 체크박스만으로는 통과할 수 없다. 작업공간 서버가 다음 상태를 판정한다.

- verified_attorney: 자격과 소속을 확인한 변호사
- operator: 기술·편집 운영자. 변호사 명의의 법률서비스를 제공할 수 없다.
- unverified: 인증은 되었으나 승인되지 않은 계정
- anonymous: 인증정보가 없는 계정

사건 목록 조회, 사건 생성, 파일·결과 열람을 포함한 모든 작업공간 화면과 API는 verified_attorney 또는 필요한 최소 범위의 operator만 허용한다. 공개 콘텐츠에서 전달되는 링크는 반드시 내부 설명 게이트인 /ko/lawyer-workspace를 먼저 거친다.

## 5. 계약·보수·책임

변호사 전용 소프트웨어는 리알레와 변호사 또는 법률사무소 사이의 고정형 이용계약으로 설계한다. 기술료는 사건 수, 상담료, 수임료, 성공 여부와 연동하지 않는다.

일반인에게 개별 법률서비스를 제공한다면 다음 주체는 변호사 또는 법무법인이다.

- 상담·위임계약의 당사자
- 보수를 받는 자
- 답변과 문서에 이름을 표시하는 자
- 이해충돌·비밀유지·기록보관을 수행하는 자
- 최종 결론을 검토·수정·거절하고 책임지는 자

## 6. 공개 데이터 계약

- concierge_entry는 공개 주제 원본과 출판 묶음에서 금지한다.
- 허용되는 연결 필드는 lawyer_workspace_entry뿐이다.
- href는 /ko/lawyer-workspace로 고정한다.
- audience는 verified_attorney로 고정한다.
- 공개 체크 상태와 사건 사실은 연결 필드에 포함하지 않는다.

조합기와 출판 검증기는 이 규칙을 자동으로 검사한다. 예전 필드나 외부 작업공간 직링크가 다시 들어오면 빌드가 실패해야 한다.

## 7. 일반인을 위한 설명 콘텐츠

게이트의 이유를 숨기지 않는다. 법률정보와 변호사 이용 주제관에서 다음을 설명한다.

- 일반 법률정보와 개별 법률상담의 차이
- 무료·광고·회원비 등 경제적 이익과 기능의 관계
- 계약·보수·명의·책임 주체의 확인 방법
- 사건 소개·알선 대가와 수임료 배분 문제
- 면책문구보다 실제 서비스 기능이 중요한 이유
- 일반인이 개별 상담을 받을 때 확인할 체크리스트

## 8. 변경 통제

공개 사이트에 사건입력, 맞춤 결론, 상담 연결, 외부 작업공간 직링크를 다시 넣으려면 별도의 법률검토와 데이터흐름 검토를 거쳐야 한다. 화면 문구만 바꾸는 방식으로 이 경계를 완화할 수 없다.
