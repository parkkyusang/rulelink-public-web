# RuleLink 공개 웹 디자인 체계

## 목적

RuleLink의 시각 품질은 장식용 이미지의 양보다 사용자가 법률정보의 위계, 최신성, 공식 근거와 다음 행동을 빠르게 구분할 수 있는지로 판단한다. 이 문서는 콘텐츠가 계속 늘어나도 홈, 검색, 지식 허브, 상세 글, 공식 근거와 법령 변화 화면이 하나의 서비스처럼 확장되게 하는 상위 계약이다.

## 현재 판단

별도 디자인 제작 세션이나 애셋 생성 세션을 상시 운영하지 않는다. 현재 우선순위는 아래 세 가지다.

1. 정보 위계와 읽기 흐름을 화면 유형 사이에서 일관되게 유지한다.
2. 색상, 글꼴, 경계선, 반경, 그림자와 입력 초점을 공통 디자인 토큰으로 관리한다.
3. 모바일에서 검색, 필터, 근거와 행동 버튼이 내용보다 먼저 깨지지 않게 검증한다.

화면군이 안정된 뒤 별도 시각 감사 세션을 열 수 있다. 그 세션의 역할은 새 콘셉트를 임의로 덧붙이는 것이 아니라 실제 주요 화면을 같은 기준으로 비교하고 불일치를 교정하는 것이다.

## 시각 방향

- 신뢰: 짙은 녹색과 크림색을 기본으로 하되, 중요한 상태는 색상만이 아니라 문구와 형태로도 구분한다.
- 근거: 기준 확인일, 원문, 법령·판례 유형과 연결 근거를 작은 보조 정보로 숨기지 않는다.
- 연역적 전개: 질문, 판단 기준, 사실분기, 행동 순서와 공식 근거가 위에서 아래로 자연스럽게 이어져야 한다.
- 절제: 사진이나 장식 삽화보다 타이포그래피, 여백, 경계선과 구조적 카드가 우선한다.
- 접근성: 키보드 초점, 충분한 대비, 동작 줄이기 설정과 모바일 한 열 구성을 유지한다.

## 코드 계약

전역 기초값은 `app/globals.css`의 `:root` 디자인 토큰으로 정의한다.

- 글꼴: `--font-body`, `--font-display`
- 본문과 보조색: `--ink`, `--muted`, `--muted-soft`, `--muted-strong`
- 표면과 경계: `--white`, `--surface-soft`, `--surface-evidence`, `--line`, `--line-strong`, `--line-hover`
- 형태와 상호작용: `--radius-card`, `--radius-control`, `--radius-pill`, `--shadow-card`, `--shadow-card-soft`, `--focus-ring`, `--focus-field`

구성요소별 스타일 파일은 위 기초값을 직접 재정의하지 않는다. 판례의 보라색, 법령 변화의 파란색처럼 의미가 분명한 상태색만 구성요소 안에 둘 수 있다.

## 화면 유형별 우선순위

| 화면 | 사용자가 가장 먼저 알아야 할 것 | 시각적으로 먼저 보여줄 것 |
| --- | --- | --- |
| 홈 | 무엇을 찾을 수 있는가 | 검색 진입, 최신 콘텐츠, 주제의 폭 |
| 전체검색 | 어떤 결과가 있는가 | 검색어, 유형 필터, 결과 수 |
| 지식 허브 | 내 상황이 어느 갈래인가 | 주제, 판단 갈래, 관련 안내 |
| 지식 상세 | 무엇이 결론을 가르는가 | 한 줄 답변, 사실분기, 확인 목록, 근거 |
| 공식 근거 | 어느 원문이 어떤 안내에 쓰였는가 | 근거 유형, 확인일, 공식 원문, 역연결 |
| 법령 변화 | 언제 무엇이 달라지는가 | 시행 상태, 날짜, 전후 규범과 행동 |

## 애셋 원칙

현재는 주제별 사진, 생성형 삽화와 장식 아이콘 묶음을 만들지 않는다. 이런 애셋은 법률정보의 신뢰를 높이기보다 화면 밀도와 유지보수 비용을 키울 가능성이 높다.

다음 조건에서는 전용 애셋 제작을 검토한다.

- 독립 도메인과 브랜드 표기가 확정되어 로고와 파비콘이 필요할 때
- 외부 공유 유입이 늘어 사회관계망 미리보기 카드가 필요할 때
- 문자만으로는 여러 지식 허브를 빠르게 구별하기 어려워 일관된 아이콘 체계가 필요할 때

그 경우에도 낱개 이미지를 즉흥적으로 만들지 않고 크기, 선 굵기, 색상, 대체 텍스트와 사용 위치를 먼저 계약으로 정한다.

## 시각 감사 통과 기준

- PC와 모바일에서 홈부터 상세·공식 근거까지 주요 경로를 실제 크기로 확인한다.
- 같은 역할의 검색창, 필터, 카드, 메타 정보와 행동 버튼이 같은 시각 언어를 사용한다.
- 긴 한글 제목, 사건번호, 조문번호와 날짜가 가로 넘침을 만들지 않는다.
- 호버 효과가 없어도 링크와 버튼을 구별할 수 있다.
- 색상 없이도 현재 필터, 시행 상태와 근거 유형을 이해할 수 있다.
- 새 핵심 색상이나 반경을 추가하면 전역 토큰 또는 명시적인 의미 색상으로 분류한다.
- 자동검증과 제작 빌드를 모두 통과한 뒤 병합한다.
